Smart greenhouse plan

План реализации умной теплицы

Цель проекта — постепенно перейти от мониторинга влажности почвы на ESP32 к полноценному гроубоксу с историей измерений, уведомлениями, автополивом, управлением светом и вентиляцией.

Архитектура

Датчики -> ESP32 -> Wi-Fi -> Ktor API -> База данных -> Панель.

MVP

Первый релиз должен быть надежным: ESP32 DevKit, 1-2 емкостных датчика влажности, отправка телеметрии на сервер и хранение измерений.

  • POST /api/v1/telemetry принимает показания.
  • Сервер хранит devices, sensors, measurements.
  • Датчики имеют калибровку сухого и влажного значения.
  • ESP32 переживает временную недоступность Wi-Fi и сервера.

Этапы

00

Проектирование: размер бокса, растения, сервер, питание.

01

ESP32, влажность почвы и первая отправка JSON.

02

Калибровка датчиков и базовый мониторинг.

03

Мини-гроубокс без силового управления от ESP32.

04

Температура, влажность воздуха, свет, вода и протечка.

05

Уведомления и правила предупреждений.

06

Автополив только после накопления истории.

07

Управление светом и вентиляцией с учетом безопасности 220V.

Ktor backend

Основной стек: Kotlin + Ktor, PostgreSQL, Flyway, kotlinx.serialization, авторизация устройств через заголовок X-Device-Token.

GET /api/v1/health healthcheck

GET /api/v1/devices список устройств

GET /api/v1/devices/{id}/latest последние показания

GET /api/v1/devices/{id}/readings история

POST /api/v1/sensors/{id}/calibration калибровка

ESP32

Прошивка хранит идентификатор устройства и локальные секреты, читает analog-значения, усредняет 5-10 замеров, применяет калибровку, отправляет JSON и пишет диагностику в Serial.

Ограничения безопасности

  • Не использовать резистивные датчики влажности для постоянной работы.
  • Не подключать 220V напрямую к ESP32.
  • Автополив включать только после истории измерений и понимания порогов.
  • Для автополива обязательны лимит времени помпы, ручной режим и датчик протечки.
