# План реализации умной теплицы / гроубокса для томатов

## 1. Цель проекта

Собрать домашнюю умную теплицу для выращивания томатов с постепенным развитием от простого мониторинга влажности почвы до полноценного гроубокса с освещением, вентиляцией, поливом, журналом показаний и серверной логикой на Ktor.

Финальная идея похожа на готовый гроубокс вроде GrowWorld `В150хД80хШ60 Quantum FR + IR + UV 240W LED SILENT`, но с собственными характеристиками, своей электроникой и серверной частью: шкаф/тент, LED-освещение, вентиляция, фильтрация, датчики, контроллер, автоматизация и веб-интерфейс.

Референс: https://growworld.ru/product/v150hd80hsh60-grouboks-quantum-fr-ir-uv-240w-led-silent

## 2. Общая архитектура

Проект делится на две большие части:

1. Теплица / гроубокс:
   - корпус, свет, вентиляция, электрика, горшки, субстрат;
   - ESP32 и датчики;
   - локальная логика сбора показаний;
   - в следующих этапах: полив, управление светом, вентиляцией и аварийные сценарии.

2. Сервер на Ktor:
   - прием телеметрии от ESP32;
   - хранение истории измерений;
   - API для графиков, статусов и настроек;
   - веб-панель или API для отдельного фронтенда;
   - правила уведомлений и автоматизации.

Базовый поток данных:

```text
Датчики -> ESP32 -> Wi-Fi -> Ktor API -> База данных -> Панель мониторинга / уведомления
```

## 3. Этапы реализации

### Этап 0. Проектирование и базовые решения

Цель: определить стартовые параметры, чтобы не покупать лишнее и не переделывать электрику.

Что решить:

- Размер будущего бокса: например, 120x60x60, 150x80x60 или другой формат под место в квартире.
- Количество растений: для томатов лучше начать с 1-2 кустов, особенно если сорт не карликовый.
- Тип выращивания на старте: почва, не гидропоника.
- Где стоит сервер: домашний сервер, VPS или локальный компьютер.
- Как ESP32 будет отправлять данные: HTTP POST на Ktor API для MVP; MQTT можно добавить позже.
- Частота отправки показаний: каждые 1-5 минут на старте, позже адаптировать.
- Формат питания: отдельное питание для ESP32 и отдельно безопасная силовая часть для света/вентиляции.

Оборудование этапа:

| Категория | Что нужно | Комментарий |
| --- | --- | --- |
| Планирование | Рулетка, схема места установки | Проверить высоту, доступ к розетке, вентиляцию помещения |
| Электрика | Сетевой фильтр с запасом по мощности | Лучше сразу с заземлением и нормальным качеством |
| Документация | Таблица параметров бокса | Размеры, мощность света, датчики, интервалы измерений |

Результат этапа:

- выбран стартовый размер;
- понятен первый набор датчиков;
- есть структура будущей системы.

### Этап 1. MVP: ESP32 + датчики влажности почвы + отправка на сервер

Цель: получить первое работающее устройство, которое измеряет влажность почвы и передает показания на сервер.

Что делает устройство:

- подключается к Wi-Fi;
- читает 1-2 датчика влажности почвы;
- усредняет несколько измерений;
- отправляет JSON на сервер;
- пишет ошибки в Serial Monitor;
- умеет пережить временную недоступность Wi-Fi или сервера.

Рекомендуемый формат JSON:

```json
{
  "deviceId": "growbox-esp32-001",
  "timestamp": "2026-05-10T12:00:00Z",
  "readings": [
    {
      "sensorId": "soil-1",
      "type": "soil_moisture",
      "raw": 2380,
      "percent": 42.5
    }
  ],
  "batteryVoltage": null,
  "rssi": -61
}
```

Оборудование этапа:

| Категория | Что нужно | Количество | Комментарий |
| --- | --- | ---: | --- |
| Контроллер | ESP32 DevKit | 1 | Лучше обычная dev board с USB-C или Micro USB |
| Датчики | Емкостной датчик влажности почвы v1.2 | 1-2 | Не брать резистивные датчики для постоянной работы: они быстро окисляются |
| Питание | USB-блок питания 5V 1-2A | 1 | Отдельный нормальный блок питания |
| Проводка | Dupont-провода, клеммники, макетная плата | Набор | Для прототипа |
| Корпус | Маленький пластиковый корпус для ESP32 | 1 | Защитить плату от влаги и земли |
| Отладка | USB-кабель | 1 | Для прошивки и Serial Monitor |

Серверная часть этапа:

- создать Ktor-приложение;
- добавить endpoint `POST /api/v1/telemetry`;
- добавить простую авторизацию по device token;
- сохранять данные в базу;
- добавить `GET /api/v1/devices/{deviceId}/latest`;
- добавить `GET /api/v1/devices/{deviceId}/readings?from=&to=`;
- заложить миграции базы.

Минимальная схема БД:

```text
devices
- id
- name
- token_hash
- created_at
- last_seen_at

sensors
- id
- device_id
- sensor_type
- name
- calibration_dry
- calibration_wet
- created_at

measurements
- id
- device_id
- sensor_id
- measured_at
- type
- raw_value
- value
- unit
- rssi
```

Результат этапа:

- ESP32 отправляет реальные значения влажности почвы;
- сервер принимает и сохраняет показания;
- можно получить последние показания через API.

### Этап 2. Калибровка влажности почвы и базовый мониторинг

Цель: превратить сырые analog values в понятные проценты и научиться видеть динамику.

Что сделать:

- замерить значение датчика в сухом воздухе;
- замерить значение в воде или полностью влажной почве;
- сохранить `calibration_dry` и `calibration_wet` для каждого датчика;
- добавить расчет процента влажности либо на ESP32, либо на сервере;
- добавить простую страницу или JSON-ответ с текущим состоянием;
- добавить график влажности за сутки/неделю;
- добавить статусы: `dry`, `ok`, `wet`, `sensor_error`.

Оборудование этапа:

| Категория | Что нужно | Количество | Комментарий |
| --- | --- | ---: | --- |
| Датчики | Дополнительный емкостной датчик влажности | 1-2 | Полезно иметь датчик на каждое растение |
| Почва | Стартовый субстрат для томатов | По объему горшков | Лучше сразу записать состав |
| Емкости | Горшки 5-10 л | 1-2 | Для карликовых сортов можно меньше, для обычных томатов больше |
| Маркировка | Бирки/наклейки для датчиков | Набор | Чтобы не путать `soil-1`, `soil-2` |

Серверная часть этапа:

- добавить хранение калибровки датчиков;
- добавить endpoint для обновления калибровки;
- добавить агрегации: среднее, минимум, максимум;
- добавить проверку выбросов и невозможных значений;
- добавить базовую документацию API.

Результат этапа:

- влажность отображается в процентах;
- понятны пороги полива для конкретной почвы;
- есть история измерений.

### Этап 3. Мини-гроубокс без автоматического управления

Цель: собрать физическую среду для томатов, но пока без силового управления от ESP32.

Ориентир по референсу:

- закрытый шкаф/бокс;
- LED-свет;
- канальная вентиляция;
- фильтр;
- отражающая внутренняя поверхность;
- таймер освещения;
- уплотнение от засветов;
- крепеж для подвеса светильника.

Для томатов стоит учитывать:

- им нужно много света;
- они активно растут в высоту;
- обычные сорта могут быстро перерасти маленький шкаф;
- лучше выбрать детерминантные, балконные или карликовые сорта для первого цикла;
- нужна хорошая вентиляция, иначе будут проблемы с влажностью, грибком и опылением.

Оборудование этапа:

| Категория | Что нужно | Количество | Комментарий |
| --- | --- | ---: | --- |
| Корпус | Гроутент или шкаф | 1 | Размер выбрать под место; референс 150x80x60, но можно свой |
| Внутренняя поверхность | Белая матовая/глянцевая пленка или майлар | По площади | Белая поверхность часто практичнее и меньше греется |
| Освещение | LED full spectrum / Quantum board | 1 | Мощность подобрать под площадь, для 60x80 см ориентир 150-240W |
| Подвес света | Тросики, цепь, карабины, крючки | Набор | Нужна регулировка высоты |
| Вентиляция | Канальный вентилятор | 1-2 | Приток и вытяжка, либо вытяжка + пассивный приток |
| Фильтрация | Угольный фильтр | 1 | Нужен, если важны запахи |
| Воздуховод | Гофра/канал 100-125 мм | 1-2 м | Диаметр согласовать с вентилятором |
| Управление светом | Розетка с таймером | 1 | На старте механический/электронный таймер проще и безопаснее |
| Электрика | Сетевой фильтр, кабель-каналы, стяжки | Набор | Навести порядок в проводах |
| Контроль климата | Термогигрометр | 1 | Дублирующий независимый прибор полезен для проверки датчиков |

Серверная часть этапа:

- добавить сущность `growbox`;
- связать устройства и датчики с конкретным гроубоксом;
- добавить ручные заметки: посадка, полив, подкормка, пересадка;
- добавить дневник выращивания.

Результат этапа:

- есть физический бокс для томатов;
- свет и вентиляция работают по простому расписанию;
- сервер уже хранит данные влажности и журнал событий.

### Этап 4. Расширение датчиков: климат, свет, вода

Цель: видеть не только почву, но и условия внутри бокса.

Что добавить:

- температуру и влажность воздуха;
- температуру почвы или воды;
- освещенность как контроль включения света;
- уровень воды в баке, если появится автополив;
- датчик протечки как страховку.

Оборудование этапа:

| Категория | Что нужно | Количество | Комментарий |
| --- | --- | ---: | --- |
| Климат | SHT31 / SHT35 / BME280 | 1 | Лучше, чем самые дешевые DHT11/DHT22 |
| Температура | DS18B20 waterproof | 1-2 | Можно использовать для почвы или бака воды |
| Свет | BH1750 или аналогичный lux sensor | 1 | Не заменяет PAR-метр, но помогает видеть включение/выключение света |
| Безопасность | Датчик протечки | 1 | Особенно важен перед автополивом |
| Уровень воды | Поплавковый датчик или ультразвуковой датчик | 1 | Для бака автополива |
| Проводка | JST-разъемы, клеммники, термоусадка | Набор | Для аккуратной сборки |

Серверная часть этапа:

- расширить типы измерений: `air_temperature`, `air_humidity`, `soil_temperature`, `light`, `water_level`, `leak`;
- добавить графики по каждому типу;
- добавить правила предупреждений;
- добавить хранение целевых диапазонов по фазам роста.

Пример целевых диапазонов для томатов:

| Параметр | Ориентир |
| --- | --- |
| Световой день | 14-16 часов на росте, 12-14 часов на плодоношении |
| Температура днем | примерно 22-26 C |
| Температура ночью | примерно 17-20 C |
| Влажность воздуха | примерно 50-70%, без долгого застоя влажного воздуха |
| Влажность почвы | подбирается по почве и сорту, важна стабильность без перелива |

Результат этапа:

- сервер показывает полную картину среды;
- можно понять, почему растение развивается хорошо или плохо;
- появляются первые автоматические предупреждения.

### Этап 5. Уведомления и правила

Цель: система сама сообщает, когда нужно вмешаться.

Правила для старта:

- влажность почвы ниже порога дольше N минут;
- влажность почвы резко не меняется после полива;
- температура выше критического значения;
- влажность воздуха слишком высокая при выключенной вентиляции;
- ESP32 не присылала данные дольше N минут;
- свет должен быть включен, но датчик света показывает темноту;
- обнаружена протечка.

Оборудование этапа:

| Категория | Что нужно | Количество | Комментарий |
| --- | --- | ---: | --- |
| Сервер | Канал уведомлений | 1 | Telegram-бот, email или push |
| Надежность | Резервный датчик температуры/влажности | 1 | Для проверки расхождений |
| Питание | UPS для сервера или роутера | Опционально | Полезно, если важна автономность |

Серверная часть этапа:

- добавить таблицу `alert_rules`;
- добавить таблицу `alerts`;
- добавить периодическую задачу проверки правил;
- добавить Telegram-уведомления;
- добавить статусы предупреждений: `open`, `acknowledged`, `resolved`;
- добавить антиспам: не слать одно и то же каждые 5 минут.

Результат этапа:

- система становится полезной без постоянного просмотра графиков;
- основные проблемы видны заранее.

### Этап 6. Автополив

Цель: перейти от наблюдения к управлению поливом.

Рекомендация: включать автополив только после того, как несколько недель собрана история влажности и понятны реальные пороги.

Логика автополива:

- поливать только если влажность ниже порога;
- учитывать минимальный интервал между поливами;
- ограничивать максимальную длительность работы помпы;
- после полива ждать стабилизации показаний;
- иметь ручной режим и аварийное отключение;
- иметь датчик протечки и контроль уровня воды.

Оборудование этапа:

| Категория | Что нужно | Количество | Комментарий |
| --- | --- | ---: | --- |
| Вода | Бак для воды | 1 | Непрозрачный, с крышкой |
| Подача воды | Мини-помпа 5V/12V или перистальтическая помпа | 1 | Перистальтическая точнее, обычная дешевле |
| Управление | MOSFET-модуль или реле | 1 | Подбирать под помпу; силовая часть отдельно от ESP32 |
| Трубки | Силиконовые трубки, капельницы, тройники | Набор | Для равномерного полива |
| Безопасность | Датчик протечки | 1 | Обязателен перед автономным поливом |
| Контроль | Датчик уровня воды | 1 | Чтобы помпа не работала всухую |
| Питание | Блок питания под помпу | 1 | С запасом по току |

Серверная часть этапа:

- добавить команды устройству: `start_watering`, `stop_watering`, `set_mode`;
- добавить таблицу `commands`;
- добавить таблицу `actuator_events`;
- добавить ручной запуск полива из панели;
- добавить расписание и правила автополива;
- хранить факт полива как событие в дневнике.

Варианты связи с ESP32:

| Вариант | Плюсы | Минусы |
| --- | --- | --- |
| HTTP polling | Просто для MVP: ESP32 сама забирает команды | Команды выполняются с задержкой до интервала polling |
| MQTT | Хорошо для IoT, удобны команды и статусы | Нужен брокер и чуть больше инфраструктуры |
| WebSocket | Можно держать живое соединение | Сложнее поддерживать на ESP32 |

Для проекта лучше начать с HTTP polling, а MQTT добавить позже, если появится несколько устройств.

Результат этапа:

- можно поливать вручную из интерфейса;
- автополив работает по безопасным правилам;
- все действия записываются в историю.

### Этап 7. Управление светом и вентиляцией

Цель: добавить управляемые исполнительные устройства, но не жертвовать безопасностью.

Важно: управление сетевыми 220V устройствами требует аккуратной сборки, изоляции, предохранителей и понимания рисков. Для старта безопаснее использовать готовые умные розетки с API или релейные модули в закрытом DIN-корпусе.

Оборудование этапа:

| Категория | Что нужно | Количество | Комментарий |
| --- | --- | ---: | --- |
| Свет | Управляемая розетка или реле для LED | 1 | Мощность с запасом |
| Вентиляция | Управляемая розетка, реле или регулятор оборотов | 1-2 | Для вытяжки и/или обдува |
| Защита | Корпус для силовой части | 1 | Никаких открытых 220V контактов |
| Защита | Предохранитель/автомат/УЗО | По схеме | Зависит от домашней электрики |
| Контроль | Датчик света | 1 | Проверка, что лампа реально включилась |
| Обдув растений | Вентилятор на клипсе | 1 | Для укрепления стебля и движения воздуха |

Серверная часть этапа:

- добавить `actuators`: свет, вытяжка, обдув;
- добавить расписание света;
- добавить ручной override с таймаутом;
- добавить журнал включений и выключений;
- добавить проверки: если свет включен по расписанию, но освещенность низкая, отправить alert;
- добавить режимы: `manual`, `schedule`, `auto`.

Результат этапа:

- свет и вентиляция управляются через сервер;
- есть расписание и история;
- система обнаруживает часть отказов.

### Этап 8. Финальный гроубокс под свои характеристики

Цель: собрать законченную версию, похожую по классу на готовый комплект, но под собственные размеры, томаты и автоматику.

Финальная комплектация может выглядеть так:

| Категория | Что нужно | Комментарий |
| --- | --- | --- |
| Корпус | Шкаф/гроутент выбранного размера | Например, 150x80x60 или больше, если место позволяет |
| Свет | LED full spectrum 150-240W+ | Под площадь и высоту растений |
| Вытяжка | Канальный вентилятор 100-125 мм | С запасом по производительности |
| Фильтр | Угольный фильтр | Если важен запах и чистый выхлоп |
| Приток | Пассивный или активный | Зависит от температуры |
| Обдув | Вентилятор внутри бокса | Для движения воздуха |
| Контроллер | ESP32 в корпусе | Лучше вывести разъемы датчиков наружу |
| Датчики | Почва, воздух, свет, вода, протечка | Минимальный набор для осмысленной автоматизации |
| Полив | Бак, помпа, трубки, капельницы | После этапа мониторинга |
| Электрика | Отдельная силовая коробка | Разделить низкое напряжение и 220V |
| Сервер | Ktor + БД + панель | API, история, графики, уведомления |

Критерии готовности:

- показатели приходят стабильно;
- есть история минимум за несколько недель;
- есть уведомления о проблемах;
- полив не может залить растение бесконечно;
- свет и вентиляция имеют ручной аварийный режим;
- вся силовая часть закрыта и аккуратно разведена;
- можно восстановить систему после перезагрузки ESP32 или сервера.

## 5. План прошивки ESP32

### MVP-функции

- хранить `deviceId`, Wi-Fi SSID, пароль и URL сервера;
- читать аналоговые значения датчика влажности;
- усреднять 5-10 замеров;
- применять калибровку;
- отправлять JSON;
- повторять отправку при ошибке;
- мигать LED или писать в Serial состояние: Wi-Fi, send ok, send failed.

### Следующие функции

- OTA-обновления;
- локальный web config portal для настройки Wi-Fi;
- watchdog;
- очередь измерений при недоступном сервере;
- поддержка нескольких датчиков;
- получение команд с сервера;
- управление помпой, светом или вентиляцией через безопасную силовую часть.

## 6. Риски и важные замечания

| Риск | Что сделать |
| --- | --- |
| Перелив томатов | Не включать автополив до накопления истории; ограничить длительность работы помпы |
| Окисление датчиков | Использовать емкостные датчики влажности, не резистивные |
| Высокая влажность и грибок | Добавить вентиляцию и движение воздуха внутри бокса |
| Перегрев от света | Измерять температуру, держать свет на регулируемом подвесе |
| Опасность 220V | Разделить силовую и низковольтную части, использовать закрытые корпуса |
| Сбой сервера | ESP32 должна продолжать локальные безопасные сценарии |
| Сбой датчика | Проверять невозможные значения и резкие скачки |

## 7. Ближайший практический план

1. Купить ESP32 DevKit, 1-2 емкостных датчика влажности почвы, провода, корпус и блок питания.
2. Поднять минимальный Ktor API с `POST /api/v1/telemetry`.
3. Настроить БД и таблицы `devices`, `sensors`, `measurements`.
4. Написать прошивку ESP32 для чтения датчика и отправки JSON.
5. Провести калибровку датчиков: сухо / влажно.
6. Добавить endpoint для последних показаний и истории.
7. Посадить первый томат в тестовый горшок и собрать 1-2 недели данных.
8. После понимания динамики влажности переходить к боксу, свету и вентиляции.

## 8. Что считать успехом первого релиза

Первый релиз не должен быть красивым, он должен быть надежным:

- ESP32 стабильно отправляет влажность почвы;
- сервер хранит измерения;
- есть калибровка датчика;
- можно посмотреть текущее значение и историю;
- система переживает перезапуск ESP32 и сервера;
- есть понятный следующий шаг к гроубоксу.
